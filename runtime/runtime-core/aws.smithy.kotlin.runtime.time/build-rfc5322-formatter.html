Build and return a DateTimeFormatter for RFC5322

tl;dr The single digit day-of-month is allowed by the spec but not desired by us.

DateTimeFormatter.RFC_1123_DATE_TIME is REALLY close to what we want but alas it will format day of the month with a single digit when < 10 instead of the fixed 2DIGIT format required by Smithy http-date timestampFormat trait (which is a specific fixed length version of RFC5322).

See also: https: *tools.ietf.org/html/rfc7231.html#section-7.1.1.1
